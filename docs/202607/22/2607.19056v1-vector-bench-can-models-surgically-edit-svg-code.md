## 摘要
基于指令的矢量编辑需要两种能力：做出所请求的修改，同时保持其他一切不变。当仅以光栅图像评判输出时，后者容易被忽视。我们提出了Vector-Bench，一个包含40个SVG修复任务的紧凑而困难的基准。每个任务配对一个损坏的SVG程序，以及作者编写的视觉指令、一个隐藏的目标程序、平均5.05个标注修复，以及平均60.55个受保护对象。指令描述可见的缺陷，而不暴露元素标识符、坐标、颜色代码或路径数据。我们定义了一个确定性的二元规范奖励：所请求的修复使用属性感知的感知容差，而未请求的渲染或应用相关结构必须保持语义不变，并且结果必须是有效的SVG。我们保留了标准的目标准确性和更严格的源保真度作为诊断指标。有效性门控的修复进展、一个近乎完整的分层，以及有效输出的非预期变更率（UCR）解释了部分结果。我们在1360个请求上评估了34个模型端点（25个列为开放权重、5个廉价对照和4个前沿封闭端点）。最强的端点仅达到15.0%的完整规范成功率，尽管平均修复进展为43.7%，这表明表面的修复进展和符合规范的编辑仍然是截然不同的。所有提示、输出、评分代码、成本以及每个任务的报告均已发布。

## Abstract
Instruction-based vector editing requires two capabilities: making a requested change and leaving everything else alone. The second is easy to miss when an output is judged only as a raster image. We introduce Vector-Bench, a compact, difficult benchmark of 40 SVG repair tasks. Each task pairs a corrupted SVG program with an author-written visual instruction, a hidden target program, 5.05 annotated repairs on average, and an average of 60.55 protected objects. Instructions describe visible defects without exposing element identifiers, coordinates, color codes, or path data. We define a deterministic binary specification reward: requested repairs use attribute-aware perceptual tolerances, while unrequested rendering- or application-relevant structure must remain semantically unchanged and the result must be a valid SVG. Canonical target equality and stricter source fidelity are retained as diagnostics. Validity-gated repair progress, a near-complete tier, and valid-output Unintended Change Rate (UCR) explain partial outcomes. We evaluate 34 model endpoints (25 listed as open-weight, 5 inexpensive controls, and 4 frontier closed endpoints) over 1360 requests. The strongest endpoint reaches only 15.0% full specification success, despite 43.7% mean repair progress, showing that apparent repair progress and specification-faithful editing remain substantially different. All prompts, outputs, scoring code, costs, and per-task reports are released.

## 1. 论文的核心问题与整体含义

- **研究动机**：当前基于指令的矢量图形编辑评测，通常仅凭生成图像（光栅化结果）进行评判，而忽略了一项关键约束——**不仅要做出所请求的修改，还必须保证未提及的其余部分完全不变**。这种评估盲区导致模型可能完成视觉上的修复，却在底层代码中改动了不希望变更的结构（如重命名元素、重写路径、移动无关对象等），对于SVG这种结构化、可执行的矢量格式而言，这些并非“表面差异”，而是实质性的破坏。
- **核心问题**：如何系统、严格地度量语言模型在**手术式矢量编辑**（surgical vector editing）上的能力，即能否**准确地仅修复指定缺陷，同时保护所有未提及元素的语义与结构完整性**。
- **整体含义**：为填补现有评测中“重修复完成、轻保护保真”的空白，论文推出Vector-Bench，旨在通过一个紧凑但高难度的基准，暴露模型在“完成部分修复”与“达成全规范忠实编辑”之间的巨大差距，并推动结构化视觉编辑模型的进步。

## 2. 论文提出的方法论

- **任务构造**：
  - 定义每个任务为元组 `τ = (x, S₀, S*, E, U)`，其中 `x` 为自然语言修复指令，`S₀` 为损坏的SVG代码，`S*` 为隐藏的目标SVG，`E` 为一组期望的对象属性修复标注，`U` 为需要保护的场景对象子集。
  - **指令设计**：仅用日常语言描述可见错误（如“琥珀色信号灯变暗了”），严禁透露DOM标识符、坐标、颜色码、路径数据等任何内部信息。
  - **数据规模**：40个密集型修复任务，包含202个标注修复（平均每任务5.05个），平均每任务60.55个受保护对象，整体文件包含294,798个字符。

- **确定性二元规范奖励**：
  - **三道关卡（gates）**：
    1. **有效性门**：输出必须是合法SVG（完整的根元素、格式正确、ID唯一、路径语法有效等）。
    2. **请求修复门**：对每个期望修复 `e ∈ E`，使用**属性感知的容差检查** `okₑ(S)`，例如颜色比较使用CIE Lab ΔE₇₆并设置容差上限，路径通过采样点进行对称最近点距离比较，任何检查都确保已知的损坏值无法通过。
    3. **语义保护门**：屏蔽所有请求修复的字段后，输出SVG的渲染及应用程序相关语义表示 `V(Mₑ(S))` 必须与目标完全相同 `V(Mₑ(S*))`，以捕捉对未请求节点、属性、元素顺序等的任何改动。
  - **最终奖励**：`Rspec(S)` 为上述三个门全部通过的与门（AND gate）。该奖励为确定性的二进制值（0或1），不依赖任何学习型裁判。
  - **诊断指标**：额外提供修复完成比例、基于容差的修复进展（进度条）、未请求更改率（UCR）、近乎完整层等分析性指标，且将源保真度（`C(S)`）和标准目标准确性作为独立诊断保留。

## 6. 论文的主要结论与发现

- **规范成功极其罕见**：在1360个结果中，仅有2.35%满足全部三道关卡。最强模型（Claude Sonnet 5）的完全规范成功率仅为**15.0%**。
- **修复进展与忠实编辑严重脱节**：模型常能完成部分可见修复（有效性门控平均修复进展27.5%，最强模型达43.7%），但绝大多数输出要么未能完成所有修复，要么产生了未预期的语义变更，导致最终奖励为零。
- **失败模式分布**：46.0%的输出为无效或缺失SVG，50.7%为有效但不完整的修复或存在附带损害，仅有0.9%完成所有修复但产生副作用。这说明瓶颈不仅在于修复能力本身，更在于**结构保真与有效性保证**。
- **修复与保护是独立轴**：修复进度与未预期更改率（UCR）之间无必然正相关，有些模型能较好修复但连带破坏了大量受保护对象。
- **任务难度与操作类型影响**：简单的颜色、插入/删除修复较易完成，但涉及路径形状、位置恢复的任务对模型构成显著挑战。

## 7. 优点

- **定义严格且可审计的编辑标准**：二元规范奖励强制要求“修复+保护+有效”，将评估从主观的“看起来像”升级为客观的结构对等检查，且所有逻辑确定、透明、可复现。
- **贴近实际编辑场景**：指令采用纯视觉语言描述，不泄露任何内部代码信息，高度模拟用户向编辑器提出修改需求的方式。
- **丰富的诊断指标体系**：除主要成功指标外，提供了修复进展、未预期更改率、有效性、按操作类型分解等分层指标，能够精细解释失败原因。
- **高度完整的开放科学实践**：公开了全部提示、原始输出、评分代码、成本以及每个任务的可追溯报告，并提供了交互式对比查看器。
- **评估器自检机制**：通过一系列确定性控制实验验证了评估器自身的行为，增强了基准的可信度。

## 8. 不足与局限

- **基准规模与泛化性**：仅包含40个任务，场景覆盖有限，属于“压力测试”而非全面能力衡量，结论可能受任务选择偏差影响。
- **缺少人类性能校准**：未进行人类一致性研究或人类表现基线测试，确定性容差是否与人类感知完全对齐未经实证。
- **采样稳定性未考虑**：每个模型-任务仅评测一次，无法反映模型输出的随机性或提供置信区间外的统计力。
- **评估依赖外部服务与版本**：API路由、定价、模型版本可能变动，虽记录了端点ID，但复现性仍受到第三方服务影响。
- **部分数据来源风险**：20个风景场景的原始文件缺乏可恢复的来源元数据，声明为“来源待定，仅限研究”，可能限制其再分发。
- **任务耦合性**：评测协议将场景理解、代码定位、修复生成综合在一起，不易分离模型在不同子能力上的瓶颈。
