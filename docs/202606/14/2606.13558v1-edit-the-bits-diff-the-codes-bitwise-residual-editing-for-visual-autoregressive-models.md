## 摘要
使用视觉自回归（VAR）生成器进行文本引导的图像编辑需要控制模型采样什么以及采样的变化写回图像编码的位置。现有的VAR编辑器主要操作于token流、特征或扁平的下一个token logits，未能充分利用按位残差VAR模型的两个原生结构：每比特伯努利预测头和组装图像所用的多尺度加性残差码场。我们提出了BitResEdit，一种针对按位残差VAR生成器（如Infinity）的无训练编辑器。BitEdit通过在共享编辑前缀上计算的源-目标对比度来倾斜分类器自由引导后的每比特对数几率，然后将每次更新投影到围绕干净CFG采样器的闭式伯努利-KL信任区域内，以此实现源负向引导。ResEdit将采样的比特转换为每个尺度的连续编码残差，用定位掩码进行门控，并通过生成器原生的尺度总和重新注入。它们共同将决策时的比特引导与组合时的编码合成耦合起来，从而通过编码算术精确保留被掩码的潜在特征，同时在目标区域内应用局部的、尺度感知的编辑。在PIE-Bench上使用Infinity-2B时，BitResEdit在同骨干VAR编辑器中获得了最强的文本对齐，编辑区域的CLIP分数比之前最强的编辑器提高了+1.07，同时背景保留能力与之相当。消融实验显示BitEdit和ResEdit在目标对齐和背景保留中发挥互补作用。

## Abstract
Text-guided image editing with visual autoregressive (VAR) generators requires controlling both what the model samples and where the sampled change is written back into the image code. Existing VAR editors mainly operate on token streams, features, or flat next-token logits, leaving two native structures of bitwise-residual VAR models underused: the per-bit Bernoulli prediction head and the additive multi-scale residual code field from which the image is assembled. We propose BitResEdit, a training-free editor for bitwise-residual VAR generators such as Infinity. BitEdit performs source-negative guidance by tilting the post-CFG per-bit log-odds along a source--target contrast computed on a shared edited prefix, then projects each update into a closed-form Bernoulli-KL trust region around the clean CFG sampler. ResEdit converts the sampled bits into per-scale continuous-code residuals, gates them with a localization mask, and re-injects them through the generator's native sum-of-scales. Together they couple decision-time bit guidance with combination-time code composition, so masked-out latent features are preserved exactly by code arithmetic while localized, scale-aware edits are applied inside the target region. On PIE-Bench with Infinity-2B, BitResEdit attains the strongest text alignment among same-backbone VAR editors, improving CLIP on the edited region by +1.07 over the strongest prior editor while keeping background preservation competitive with it. Ablations show BitEdit and ResEdit play complementary roles in target alignment and background preservation.

### 2. 论文提出的方法论
论文的核心思想是“编辑比特，差异编码”，将编辑过程分解为两个相互耦合的阶段，直接作用于VAR生成器的原生运算节点。

*   **第一阶段：BitEdit（比特编辑）—— 决策时的对数几率引导**
    *   **核心思想**：在模型采样前，对每个位置、每层尺度的**每比特对数几率（log-odds）** 进行精确引导，使采样结果偏向目标提示词，同时偏离源提示词。
    *   **关键技术细节**：
        1.  **三路前向传播**：在一个共享的编辑前缀（`˜c<k`）下，计算三个条件下的每比特对数几率：目标提示（T）、无条件（∅）和源提示（S）。
        2.  **源-负对比度**：计算一个源-目标的每比特对比方向 `∆k = dk(S) − dk(T)`，用于将采样结果推离“源”状态。
        3.  **信任区域投影**：将原始引导后的对数几率 `d_raw` 投影到**伯努利-KL信任区域**内，以确保编辑后的采样分布不会偏离干净的分类器自由引导（CFG）采样器过远。具体是通过二分查找找到最大混合权重 `λ⋆`，使得 `KL(Bern(σ( d_guided)) || Bern(σ( d_CFG))) ≤ δKL`。
        4.  **尺度退火策略**：引导强度 `η_k` 从最粗尺度到最细尺度线性退火（`η_k = η_0 * (1 - k/K)`），在粗尺度（布局/语义）施加最强引导，在细尺度（纹理）减弱，以防止破坏高频细节。

*   **第二阶段：ResEdit（残差编辑）—— 组合时的码空间注入**
    *   **核心思想**：将BitEdit采样得到的离散比特转化为连续码，计算其与源图像码之间的残差，然后通过一个定位掩码进行空间选通，最后将选通后的残差**精确地加回**到VAR生成器的原生多尺度求和中。
    *   **关键技术细节**：
        1.  **残差计算**：在最终的VAE潜在空间分辨率上，计算编辑采样码（`˜c_k`）和缓存的源图像码（`c_src_k`）之间的残差 `∆c_k`。
        2.  **掩码选通注入**：最终的编辑码由公式 `c_out_k = c_src_k + α * G_final ⊙ ∆c_k` 确定。
            *   `G_final` 是经过膨胀、模糊和峰值归一化处理的软性定位掩码，决定编辑生效的区域。
            *   `α` 是残差强度标量。
        3.  **算术保真性**：在掩码区域外（`G_final=0`），等式退化为 `c_out_k = c_src_k`，意味着通过代码空间的代数运算，背景潜在特征被**完整保留**，变化仅发生于掩码内部。

*   **耦合机制**：BitEdit（对数几率空间）和ResEdit（码空间）通过累积的编辑潜在码 `˜z≤k` 耦合，该潜在码被下采样后，作为下一尺度的视觉前缀输入，实现尺度间的信息传递与协同编辑。

### 8. 不足与局限
*   **对掩码质量的依赖**：文中明确指出，该方法高度依赖输入的二值定位掩码。当掩码不准确（如使用边界框替代）时，编辑效果会显著下降。编辑质量是“相对于提供掩码的保证”。
