## 摘要
近年来，统一视觉-语言模型在单一架构中处理多模态理解和生成方面取得了显著进展。自回归视觉语言模型虽然能够跨模态推理，但无法生成高质量图像。相比之下，扩散模型能生成逼真的视觉内容，却难以生成连贯的文本，这使得开发一个既能无缝处理视觉生成又能处理文本生成的统一模型充满挑战。最新研究表明，语言可以有效地嵌入到视觉表示中，使模型能够直接从图像中推理文本语义。为此，我们提出UniCanvas，这是首次尝试通过图文生成统一扩散模型以生成交错多模态内容。扩散模型自然地捕获共享像素画布上的变换，可视为视觉变化的世界模型。该模型不是生成离散文本标记，而是通过学习将语言表示为图像中的视觉模式，利用其内在的多模态嵌入空间。这一设计使模型在图像合成过程中能够在单个像素画布上自然地“绘制”文本，实现无缝的多模态生成。实验表明，UniCanvas相比之前的统一模型性能提升，将基于扩散模型的图文生成定位为一种有前途的统一多模态生成范式。

## Abstract
Recent years have seen remarkable progress in unified vision-language models handling both multimodal understanding and generation within a single architecture. While autoregressive VLMs can reason across modalities, they fail to generate high-quality images. In contrast, diffusion models produce photorealistic visuals yet struggle to generate coherent text, making it challenging to develop a single unified model that can seamlessly handle both visual and text generation. Recent advances suggest that language can be effectively embedded within visual representations, allowing models to reason about textual semantics directly from images. To this end, we propose UniCanvas, a first attempt that unifies diffusion models to generate interleaved multimodal contents through text-in-image generation. Diffusion models naturally capture transformations on a shared pixel canvas, which can be viewed as world models of visual change. Instead of producing discrete text tokens, the model learns to represent language as visual patterns inside images, leveraging its inherent multimodal embedding space. This design allows the model to "draw" text naturally within a single pixel canvas during image synthesis, achieving seamless multimodal generation. Experiments demonstrate that UniCanvas improves performance over previous unified models, positioning text-in-image generation with diffusion models as a promising unified multimodal generation paradigm.

### 2. 方法论：核心思想与关键技术

*   **核心思想**：UniCanvas 提出了一个**纯扩散框架**，所有操作都在一个共享的像素潜变量空间（writable latent space / pixel-native canvas）上进行。模型学习通过扩散过程，在画布上“写入”文本（作为视觉结构），并“更新”画布状态以反映场景变化。文本不再以离散 token 表示，而是像素渲染的符号。
*   **关键技术细节**：
    1.  **文本-图像生成 (Text-in-Image Generation)**：采用**流匹配 (Flow Matching)** 公式。给定输入图像 \( x \) 和目标文本图像 \( x_{text}^0 \)，训练一个速度网络 \( v_\theta \) 来预测线性插值路径 \( x_t = (1-t)x_0 + tx_1 \) 的瞬时速度 \( v_t = x_1 - x_0 \)。这使模型学会在保持图像整体结构的同时，在画布上绘制光学文本。
    2.  **两阶段生成 (Two-Stage Generation)**：为将推理（reasoning）与渲染（rendering）解耦，提出两阶段流程，共享同一个扩散主干网络 \( v_\theta \)：
        *   **阶段1：视觉推理 (Stage 1: Visual Reasoning)**：模型根据推理提示 \( c_{reason} \)，通过流匹配将当前图像画布 \( x \) 转换为增强推理的画布 \( \hat{x}_{text}^0 \)，该画布包含语义文本（如动作标签）。
        *   **阶段2：视觉合成 (Stage 2: Visual Synthesis)**：模型根据执行提示 \( c_{exec} \)，通过流匹配将推理增强画布 \( \hat{x}_{text}^0 \) 转换为下一状态场景图像 \( \hat{x}_{state}^0 \)。
        *   多步任务通过迭代执行“阶段1 → 阶段2 → 将输出作为新输入”的流程实现。
    3.  **CLIP 对齐正则化 (CLIP-based Alignment Loss)**：添加一个轻量级对比正则项 \( L_{CLIP} \)，利用预训练、冻结的 CLIP 模型，鼓励中间推理画布 \( \hat{x}_{text}^0 \) 的视觉嵌入与对应文本提示 \( c_{reason} \) 的文本嵌入在语义上对齐，从而提高生成光学文本的**可读性和语义一致性**。
    4.  **预训练 (Pre-train with Perceptual Loss)**：为解决现有扩散模型无法生成图像内文本的问题，引入预训练阶段。将 VQA 数据中的答案渲染为目标图像中的光学文本，并混合通用图像编辑数据。除 CLIP 损失外，还添加了基于 LPIPS 的感知损失 \( L_{perc} \)，专门在文本区域上稳定字体渲染。
*   **总训练目标**：\( L_{total} = L_{text}^{FM} + L_{state}^{FM} + \lambda_{CLIP} L_{CLIP} \)。

### 7. 优点

*   **创新性**：首次提出了**纯扩散模型实现图文联合生成**的范式，摆脱了对自回归 token 解码和外部 VLM 的依赖。将文本视为一种视觉模式进行生成，从根本上弥合了语言与视觉之间的表征鸿沟。
*   **方法论清晰**：两阶段生成（推理+渲染）的设计简洁有效，将复杂的多步任务分解为更可控的子问题，并共享同一个扩散骨干网络。
*   **实验扎实**：在多个具有挑战性的多步推理基准（如 VSP, Recipe, RLBench）上进行评估，消融实验全面，对比基线强大，验证了方法有效性。

### 8. 不足与局限

*   **长文本生成能力弱**：对于需要生成多句解释的长文本推理任务，UniCanvas 表现不可靠，其底层骨干网络更适合局部文本渲染。
*   **视觉误差累积**：多步生成长度增加时，视觉合成误差（如状态预测错误、文本渲染不清）会累积，导致轨迹偏离，最终失败（附录 Fig.6, 7）。

*   **实验范围**：主要实验集中在特定任务（导航、烹饪、简单机械臂操作），在更复杂的开放域或需要大量常识推理的任务上尚未验证。性能提升的普适性有待进一步探索。
