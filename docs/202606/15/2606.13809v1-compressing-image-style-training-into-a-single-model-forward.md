## 摘要
基于扩散的风格迁移必须在推理效率与风格化保真度之间取得平衡。基于适配器的方法效率高，但将风格作为外部条件注入，可能削弱参考图像特有的外观，或将参考图像的语义内容复制到生成图像中。基于优化的个性化方法（如 LoRA）能更有效地内化风格，但每个新风格都需要单独的训练过程。我们提出 i2L（image-to-LoRA），一个将风格 LoRA 训练摊销为单次前向传播的框架。给定一张或多张参考图像，i2L 为文本到图像模型预测 LoRA 权重，无需逐风格优化即可立即实现风格实例化。该架构结合了图像编码器、可学习的 LoRA 查询以及生成适配矩阵的压缩解码头。在语义多样的风格对上训练促使预测器保留外观线索，同时抑制参考内容复制。在 Z-Image、FLUX.2 和 Hidream-O1 上的实验表明，i2L 在风格保真度、提示对齐和感知质量上均优于现有基线。由于 i2L 生成显式的 LoRA 权重，它还支持非对称无分类器引导、多参考风格融合以及与可控生成模块的组合。

## Abstract
Diffusion-based style transfer must balance inference efficiency with stylization fidelity. Adapter-based methods are efficient, but they inject style as an external condition and can either weaken reference-specific appearance or copy reference semantics into the generated image. Optimization-based personalization methods such as LoRA internalize style more effectively, but require a separate training process for every new style. We introduce i2L (image-to-LoRA), a framework that amortizes style LoRA training into a single forward pass. Given one or more reference images, i2L predicts LoRA weights for a text-to-image model, enabling immediate style instantiation without per-style optimization. The architecture combines an image encoder, learnable LoRA queries, and compressed decoding heads that generate adapted matrices. Training on semantically diverse style pairs encourages the predictor to preserve appearance cues while suppressing reference-content copying. Experiments on Z-Image, FLUX.2, and Hidream-O1 show that i2L improves style fidelity, prompt alignment, and perceptual quality over existing baselines. Because i2L produces explicit LoRA weights, it also supports asymmetric classifier-free guidance, multi-reference style fusion, and composition with controllable-generation modules.

## 1. 论文的核心问题与整体含义
扩散模型在图像风格迁移中展现出强大潜力，但现有方案陷入效率与保真度的两难：
- **适配器方法**（如 IP‑Adapter、ControlNet）效率高，一次推理即可用，但风格作为外部条件注入，容易出现风格保真度弱、语义泄漏或内容复制等问题。
- **基于优化的个性化方法**（如 LoRA 微调）能更好地将风格内化成模型权重，但每个新风格都需要反复训练，计算成本高、时间慢，不适合交互式或大规模部署。

本工作试图将 LoRA 的逐风格训练过程“压缩”到**单次模型前向传播**中，提出 **i2L（image‑to‑LoRA）** 框架：给定参考图像，直接预测适配文生图模型的 LoRA 权重，从而在无需额外训练的前提下，即时获得风格专用的生成器。

## 2. 方法论

### 核心思想
- 将“从参考图到风格 LoRA”建模成一个可学习的映射，由冻结的扩散模型反馈信号端到端训练，用一次前向替代原需数百步的风格 LoRA 微调。
- 输出的是**显式 LoRA 矩阵**，而非辅助条件特征，使其兼具适配器的推理效率和 LoRA 的内化表达力。

### 关键架构设计
1. **图像编码**：使用冻结的 SigLIP2 提取每个参考图像的块级嵌入（保留局部风格线索），多张参考图直接拼接成 `Z_img`。
2. **LoRA 查询机制**：为每层、每个秩分配可学的查询向量 `q_{Aℓ,m}` 和 `q_{Bℓ,m}`，分别对应 LoRA 矩阵的行和列。若 LoRA 秩为 k，每层共 2k 个查询。
3. **Transformer 聚合**：将图像 token 与所有 LoRA 查询拼接送入**单流 Transformer**，让查询通过自注意力同时关注图像信息及其他查询，以协调多层、多秩的权重更新。
4. **压缩线性解码**：查询的输出状态 `h` 先经低维压缩线性层 `C`，再经扩展层 `D`，生成对应 LoRA 行/列：
   ```
   A_ℓ[m,:] = D_A^ℓ C_A^ℓ h_{Aℓ,m}
   ```
   类似生成 `B_ℓ` 的列。这样可大幅降低每层独立解码头的参数量，适应现代扩散 Transformer 大量适配层的需求。

### 训练目标
- 采用标准 **Flow Matching** 损失，固定基础扩散模型和图像编码器，只训练 i2L：
  ```
  L_FM = E[ || v_{θ+ΔΘ_R}(x_t, t, c) − u_t ||_2^2 ]
  ```
  其中 `ΔΘ_R` 为从参考图 `R` 预测的 LoRA，梯度通过 LoRA 作用反向传播。

### 数据构造与防泄漏
- 使用 **MegaStyle‑1M** 数据集，它提供风格一致但内容差异大的风格‑内容对，训练时目标图与参考图风格相同、内容不同，促使预测器学习纯风格线索，抑制语义复制。

### 非对称 LoRA 引导
- 在无分类器引导中，正分支用参考图的 LoRA，负分支用**纯灰图**预测的“中性” LoRA，产生对风格特化方向的增强，进一步提升风格保真度，无需额外训练。

## 6. 主要结论与发现
- i2L 能够将**风格 LoRA 训练压缩为单次前向**，在三个主流扩散模型上均取得**更高的风格保真度、更好的提示对齐和感知质量**。
- 相比适配器方法，i2L 有效减少了语义泄漏和内容复制；相比需优化微调的 LoRA 方法，它提供了“即时”风格化能力。
- 预测的显式 LoRA 可自然支持**多参考风格融合**（无需后处理融合）和与**可控生成模块的组合**，无需修改原模型接口。
- 非对称 LoRA 引导通过正负分支应用不同 LoRA 进一步强化风格效果，且没有任何额外训练开销。

## 7. 优点
- **方法简洁高效**：将繁琐的微调压缩为一次前向，既快又好。
- **架构设计精巧**：LoRA 查询对齐行/列结构，压缩解码头控制参数规模，适合大规模扩散 Transformer。
- **数据驱动防泄漏**：利用 MegaStyle‑1M 的内容‑风格解耦数据训练，减轻物体/身份复制问题。
- **高可组合性**：输出的标准 LoRA 可直接与其他控制模块（ControlNet、AttriCtrl、Inpainting）插拔使用。
- **代码与模型开源**（三个基座模型的 i2L 权重和 DiffSynth‑Studio 代码），便于复现和社区扩展。
- **轻量推理**：风格推理只需一次编码 + 一次 LoRA 预测，成本远低于微调。

---

## 8. 不足与局限
- **依赖特定训练数据**：训练集 MegaStyle‑1M 虽然风格多样，但若目标风格明显偏离其分布，预测质量可能下降；文章未测试极端域外风格。
- **基座模型绑定**：i2L 针对特定扩散骨干训练，换用其他模型需要重新训练预测器；未验证跨模型泛化。
- **负分支中性 LoRA 设计的通用性**：用纯灰图作为“中性”参考，是否对所有风格都足够中性？可能在某些情况下引入偏差。
- **计算训练成本**：每个基座需 7 天 × 8 A100，训练复现门槛较高，且未给出更轻量的版本。
- **评估维度**：虽多指标但仍无人类主观评测；风格保真度的“CLIP‑Style”依赖 CLIP 空间，可能不完全反映人的风格感知。
- **视频/3D 扩展未提及**：论文仅验证静态图像生成，未探讨在视频风格化、3D 纹理迁移等任务上的应用潜力。
- **消融细节有限**：对 Transformer 层数、查询数量、压缩维度等超参数未作详细分析。
