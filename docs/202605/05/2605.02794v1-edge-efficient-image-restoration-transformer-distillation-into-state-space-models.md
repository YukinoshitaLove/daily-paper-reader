## 摘要
我们提出了一种用于混合图像修复的模块化框架，将 Transformer 和状态空间模型（State-Space Model, SSM）模块相结合，重点提升边缘硬件上的运行效率。虽然 Transformer 通过自注意力机制提供了强大的全局建模能力，但其注意力核在移动设备上处理高分辨率输入时会带来显著的延迟。相比之下，诸如 Mamba 等 SSM 能够实现线性时间的序列建模，具有更低的运行开销，但在细粒度修复任务上表现可能欠佳。为平衡精度与效率，我们将轻量级的 SSM 模块训练为 Transformer 模块的特征蒸馏替代体，并利用它们构建混合的 U-Net 式架构。为自动发现有效的模块组合，我们引入了高效网络搜索（Efficient Network Search, ENS），这是一种多目标搜索策略，可从预对齐的组件中选择针对任务的混合配置。ENS 在优化修复质量的同时惩罚 Transformer 的使用，以作为延迟的轻量代理，实现无需反复进行硬件分析的架构搜索。在 Snapdragon 8 Elite CPU 上，Restormer 基线推理时间为 10119.52 毫秒；相比之下，ENS 发现的混合模型显著缩短了运行时间：ENS-Deblurring 推理时间为 2973 毫秒（快 3.4 倍），ENS-Deraining 为 5816 毫秒（快 1.74 倍），ENS-Denoising 为 8666 毫秒（快 1.17 倍），同时保持了具有竞争力的修复质量。

## Abstract
We propose a modular framework for hybrid image restoration that integrates transformer and state-space model (SSM) blocks with a focus on improving runtime efficiency on edge hardware. While transformers provide strong global modeling through self-attention, their attention kernels incur substantial latency on mobile devices, especially for high-resolution inputs. In contrast, SSMs such as Mamba offer lineartime sequence modeling with lower runtime overhead but may underperform on fine grained restoration tasks. To balance accuracy and efficiency, we train lightweight SSM blocks as feature-distilled surrogates of transformer blocks and use them to construct hybrid U-Net-style architectures. To automatically discover effective block combinations, we introduce Efficient Network Search (ENS), a multi-objective search strategy that selects task-specific hybrid configurations from pre-aligned components. ENS optimizes restoration quality while penalizing transformer usage, serving as a lightweight proxy for latency and enabling architecture discovery without repeated hardware profiling. On a Snapdragon 8 Elite CPU, the Restormer baseline requires 10119.52 ms for inference. In contrast, ENS-discovered hybrids significantly reduce runtime: ENS-Deblurring runs in 2973 ms (3.4x faster), ENS-Deraining in 5816 ms (1.74x faster), and ENS-Denoising in 8666 ms (1.17x faster), while maintaining competitive restoration quality.

- **研究动机**：  
  Transformer 架构在图像修复任务中表现优异，得益于其自注意力机制对全局依赖的建模能力。然而，其 **二次复杂度 (quadratic complexity)** 在高分辨率输入和移动端设备上引发 **显著延迟和高推理成本**。  
  相比之下，状态空间模型（State-Space Models, SSM），尤其是 **Mamba 系列**，具备线性复杂度与更好的硬件适配性，但在**细粒度结构恢复**任务上往往不如 Transformer 精细。  
  本文旨在**结合 Transformer 的表达能力与 SSM 的推理效率**，探索 **在边缘设备上实现高效图像修复** 的可能性。

- **研究问题的本质**：  
  如何在不显著损失修复质量的前提下，有效减少 Transformer 模型在移动端设备上的推理延迟？  
  → 提出通过知识蒸馏将 Transformer “特征级表示能力”转移至 SSM，从而实现模块替代与结构混合优化。

## 二、方法论与核心技术

### 1. 核心思想
- 采用 **模块化混合架构**：在典型的 U-Net 框架中，交替使用 Transformer 块（以 Restormer 为代表）和 SSM 块（以 MambaIR 为代表）。  
- 通过 **特征级知识蒸馏（Feature-wise Knowledge Distillation, FwKD）**，使 MambaIR 成为 Transformer 的“轻量数字孪生”（Digital Twin），从而可直接替换部分 Transformer 模块。  
- 使用 **高效网络搜索（Efficient Network Search, ENS）**，通过多目标优化策略在精度与复杂度之间寻找最优折中，实现无需硬件重测的搜索。

### 2. 关键技术流程
- **(1) Distillation 蒸馏阶段**：  
  对每个 Restormer 模块单独训练对应的 MambaIR 模块，使其输出特征 \( O_S \) 接近教师模型输出 \( O_T \)，最小化损失函数  
  \[
  \mathcal{L}_{distill} = \| O_S - O_T \|_2^2
  \]  
  → 得到一组已对齐的轻量模块，用于后续架构组合。

- **(2) Hybrid 架构搜索阶段**：  
  建立 8 维混合搜索空间（对应 U-Net 的不同层级，每层候选为 Transformer 或多级 MambaIR 模块数量组合），形成 34,560 种可能架构。  
  ENS 以 **Bayesian Optimization + Expected Hypervolume Improvement (EHVI)** 为核心，优化：
  - 目标一：最大化 PSNR。
  - 目标二：最小化 Transformer 数量（即复杂度惩罚）。  
  最终进行 **Pareto 前沿分析 + 膝点选择（Knee point selection）** 来确定最优模型。

- **(3) Fine-tune 阶段**：  
  对搜索到的混合架构进行端到端微调，以弥补蒸馏替代导致的特征失配。

## 八、不足与局限

- **替代假设的局限**：  
  蒸馏过程假设单块 SSM 可以完全拟合 Transformer 的功能，但在深层次交互依赖上仍存在缺失。

- **复杂度代理非直接**：  
  ENS 使用 Transformer 数量作为延迟的间接度量，可能与实际设备差异存在偏差。

- **未涵盖自适应卷积或混合任务**：  
  研究集中于恢复任务，对生成或语义相关任务（如去雾、超分辨率）验证不足。

- **协同训练缺失**：  
  模块替换过程中未充分考虑互适应性，仅通过微调恢复性能，可能影响最终上限。
