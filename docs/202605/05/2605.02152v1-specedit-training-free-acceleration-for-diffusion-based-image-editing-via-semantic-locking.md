## 摘要
基于扩散模型的图像编辑在语义可控性方面表现出色，但由于需要对所有空间 token 进行迭代的高分辨率去噪，计算成本依然很高。动态分辨率采样通过在早期步骤降低分辨率可以减少这种开销。然而，现有方法通常基于低层次启发式策略（如边缘检测或通道方差）进行上采样，这些方法与编辑语义的关联较弱，可能导致结构不一致。此外，空间区域往往在没有验证是否需要语义修改的情况下被上采样，造成冗余的高分辨率计算和误差累积。因此，我们提出了 SpecEdit，一种针对扩散模型图像编辑设计的无需训练的动态分辨率框架。SpecEdit 采用“草稿-验证”（draft-and-verify）方案：首先通过低分辨率草稿估计语义结果，然后利用 token 级差异识别与编辑相关的 token 进行高分辨率去噪，而其余 token 保持粗分辨率。在 Qwen-Image-Edit 和 FLUX.1-Kontext-dev 上的实验表明，SpecEdit 实现了最高 10 倍和 7 倍的加速，同时保持高图像质量。SpecEdit 可与 step distillation 等其他加速技术互补，结合后加速可达 13 倍。我们的代码包含在补充材料中，并将于 GitHub 发布。

## Abstract
Diffusion-based image editing offers strong semantic controllability, but remains computationally expensive due to iterative high-resolution denoising over all spatial tokens. Dynamic-resolution sampling reduces this cost by performing early steps at reduced resolution. However, existing approaches prioritize upsampling using low-level heuristics such as edge detection or channel variance, which are weakly aligned with editing semantics and may lead to structural inconsistency. Moreover, spatial regions are often upsampled without verifying whether semantic modification is actually required, resulting in redundant high-resolution computation and accumulated errors. Therefore, we propose SpecEdit, a training-free dynamic-resolution framework tailored for diffusion-based image editing. SpecEdit follows a draft-and-verify scheme: a low-resolution draft first estimates the semantic outcome, after which token-level discrepancies are used to identify edit-relevant tokens for high-resolution denoising, while the remaining tokens stay at a coarse resolution. Experiments on Qwen-Image-Edit and FLUX.1-Kontext-dev demonstrate up to 10x and 7x acceleration, while maintaining strong quality. SpecEdit is complementary to step distillation and other acceleration techniques, achieving up to 13x speedup when combined with existing methods. Our code is in supplementary material and will be released on GitHub.

  - 对真正语义变化区域忽视；
  - 全局结构不稳定或语义不一致。

## 二、方法论：SpecEdit 框架

### （1）总体思想
- **核心理念**：以“草图-验证”（*draft-and-verify*）的方案实现语义驱动的动态分辨率推理。  
- **关键目标**：通过一个超低分辨率编辑“草图”预估语义变更区域，再仅对这些区域执行高分辨率去噪。

### （2）模块流程
1. **草图阶段（Draft Stage）**  
   - 对输入潜变量 \( z_{ori} \) 进行 **16× 下采样**，得到低维 latent；
   - 执行完整的低分辨率去噪过程，生成粗略编辑结果 \( \tilde{z}_{draft} \)；
   - 计算草图与原输入在感知特征空间（多层次编码器特征）中的差异 \( S(i,j) \)。

2. **语义验证（Semantic Verification）**  
   - 使用归一化的多层特征距离：
     \[
     S(i,j)=\frac{1}{L}\sum_{\ell=1}^L \| \text{Norm}(\Phi_\ell(U_s(\tilde{z}_{draft}))_{i,j} - \text{Norm}(\Phi_\ell(z_{ori}))_{i,j})\|^2
     \]
   - 当 \( S(i,j)>\tau \) 时，将该 token 视为需修改的 **语义相关区域 \( T_{edit} \)**。

3. **动态分辨率采样（Dynamic-Resolution Sampling）**
   - 主推理阶段对整个 latent 进行 **4× 下采样**；
   - 将 \( T_{edit} \) 的 token 扩展至高分辨率（2×2的 token 区块），并加入少量均匀分布的背景 token \( T_{uniform} \) 以维持全局一致性；
   - 仅对这些扩展区域执行完整 Transformer 计算，其余区域仅保持粗分辨率。

4. **最终还原**
   - 所有 token 的空间分辨率恢复至全尺寸，生成最终结果。

### （3）技术特点
- **无需额外训练**（training-free）：使用原模型权重；
- **混合分辨率计算**：不同 token 在同一网络中以不同分辨率参与；
- **兼容性强**：可叠加 step distillation、feature caching 等时间域加速措施；
- **语义锁（Semantic Locking）机制**：确保仅受影响的区域被局部更新，未编辑区域保持原貌。

- **稳健性优**：在不同任务、不同分辨率下表现一致；
- **代价低廉**：低分辨率草图阶段计算量很小，相比完整去噪几乎可忽略。

## 八、不足与局限

- **阈值选择敏感性**：语义验证阈值 τ 需手动设定，过低或过高均影响置信度；
- **草图分辨率限制**：过度下采样（>16×）会导致语义识别不准确；
- **主要针对图像编辑任务**：尚未验证在视频或大规模文本引导生成中的表现；
- **未涉及训练加速**：SpecEdit 是推理阶段优化，无法减少模型训练成本；
- **资源描述不详**：缺少关于使用 GPU 数量与耗时的详细报告；
- **极端编辑难题**：对于全局风格或背景大改的指令，其加速效益下降（语义区域过大）。
