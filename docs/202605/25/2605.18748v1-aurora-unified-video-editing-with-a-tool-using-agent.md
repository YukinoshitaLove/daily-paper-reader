## 摘要
近期视频编辑模型已收敛于一种统一的条件设计：单个扩散变换器联合处理文本、源视频和参考图像，且一组权重即可涵盖替换、移除、风格迁移和参考驱动的插入。该设计灵活，但假定用户已提供模型就绪的文本、参考图像以及局部编辑的空间定位，而实际请求往往缺失这些。我们提出Aurora，一个将工具增强的视觉语言模型（VLM）代理与统一视频扩散变换器相结合的代理式视频编辑框架。VLM代理将原始用户请求映射到与变换器条件通道对齐的结构化编辑计划，从而在生成前解决文本和视觉上的欠指定问题。我们使用监督数据训练VLM代理进行完整的编辑规划和参考图像选择，并结合偏好对以实现鲁棒的工具使用和指令精炼。我们引入AgentEdit-Bench，用于在文本和视觉欠指定下评估代理增强的视频编辑。在AgentEdit-Bench和两个现有视频编辑基准上的实验表明，Aurora相较于仅依赖指令的基线有所提升，且VLM代理可迁移至兼容的冻结视频编辑模型。项目页面：https://yeates.github.io/Aurora-Page

## Abstract
Recent video editing models have converged on a unified conditioning design: a single diffusion transformer jointly consumes text, source video, and reference images, and one set of weights covers replacement, removal, style transfer, and reference-driven insertion. The design is flexible, but it assumes that the user already provides model-ready text, reference images, and spatial grounding for local edits, which real requests often omit. We present Aurora, an agentic video editing framework that pairs a tool-augmented vision-language model (VLM) agent with a unified video diffusion transformer. The VLM agent maps a raw user request to a structured edit plan aligned with the transformer's conditioning channels, thereby resolving textual and visual underspecification before generation. We train the VLM agent with supervised data for complete edit planning and reference-image selection, together with preference pairs for robust tool use and instruction refinement. We introduce AgentEdit-Bench to evaluate agent-enhanced video editing under textual and visual underspecification. Experiments on AgentEdit-Bench and two existing video editing benchmarks show that Aurora improves over instruction-only baselines and that the VLM agent transfers to compatible frozen video editing models. *   **研究动机**：当前领先的视频编辑模型趋向于“统一条件设计”，即一个模型同时接收文本指令、源视频、参考图像和掩码等多种条件来完成编辑。然而，这种设计假设用户能提供所有这些完备的“模型就绪”条件。
*   **核心问题**：现实中，用户请求往往是“欠指定”的，具体表现为：
    *   **视觉欠指定**：指令中提到了特定实体（如“博柏利格子围巾”），但未提供对应的参考图像。
    *   **文本欠指定**：指令过于模糊或间接（如“把蓝色的东西换成跑得更快的交通工具”），需要模型自行推断具体的编辑目标。
*   **整体含义**：论文指出，当前视频编辑的瓶颈已从“模型能接受什么条件”转变为“如何将用户的原始请求转化为模型所需的条件”。Aurora 框架旨在通过引入一个智能“代理”，在生成前自动补全这些缺失的条件，从而弥合这一鸿沟。

### 2. 论文提出的方法论
Aurora 框架的核心思想是**解耦条件构建与视频生成**，由一个工具增强的 VLM 代理负责前者，一个统一的视频扩散变换器负责后者。

*   **核心思想与架构**
    *   **VLM 代理**：基于 Qwen3-VL-8B-Instruct，通过 LoRA 微调，负责理解原始请求和源视频，输出一个结构化的**编辑计划**。
    *   **视频扩散变换器**：基于 Wan2.2-TI2V-5B，专门消费由 VLM 代理构建的完备条件元组，并生成编辑后的视频。两部分权重不共享。

*   **关键技术细节与流程**
    *   **VLM 代理的编辑计划**：将原始请求映射为一个四字段 JSON 计划：
        1.  **重写指令 $(y')$**：将模糊请求具体化、标准化。
        2.  **任务标签 $(c)$**：判别编辑类型（如替换、添加、背景更改），用于指导重写，但不传给视频模型。
        3.  **图像搜索查询 $(q)$**：当需要参考图像时，生成检索词。
        4.  **掩码短语 $(m)$**：当需要空间定位时，生成用于分割的文本描述。
    *   **工具执行与条件构建**：
        *   **网络图像搜索工具**：当 $q$ 非空时，调用 Serper API 检索图像，并由代理从中选择最合适的一张作为参考 $R^+$。
        *   **分割工具**：当 $m$ 非空时，调用 GroundingDINO 和 SAM 生成掩码，并在源视频上合成被掩码的图像，将其作为特殊参考 $R^+$。
        *   最终，构建出统一条件元组 $\mathbf{x} = (y', V_{src}, R^+)$，其中 $V_{src}$ 是源视频，$R^+$ 是合并了所有参考的集合。
    *   **视频 DiT 的统一条件机制**：
        *   **多模态上下文编码器**：使用冻结的 Qwen3.5-4B 将重写指令、采样的源视频帧和参考图像编码为统一的上下文标记，注入 DiT 的交叉注意力层。
        *   **潜在标记序列与零时间步调制**：在标记序列维度，将带噪视频令牌、源视频令牌和参考图像令牌直接拼接。在 DiT 的 AdaLN 调制中，对源和参考令牌始终施加“零时间步”调制，使其不受去噪时间步影响，从而更好地保留内容。

*   **训练流程**
    *   **视频 DiT 训练**：使用标准流匹配目标，混合图像编辑、指令视频编辑和参考引导视频编辑的数据进行训练。
    *   **VLM 代理 SFT**：使用构造的数据，监督代理学习生成完整的编辑计划和从候选集中选择正确的参考图像。
    *   **VLM 代理偏好对齐 (DPO)**：针对五种边界情况（如误触发搜索、模糊掩码、约束丢失等），使用 DPO 对齐代理的决策，使其更精确。

    *   **主要基线**：UniVideo、Kiwi-Edit，以及其他开源/闭源模型如 Runway Aleph、VACE、Ditto 等。
    *   **对比维度**：不仅对比 Aurora 完整框架与基线的原始性能（无代理），还测试了“将 Aurora VLM 代理的输出提供给基线模型”的增益，以分离代理的贡献。
*   **评价指标**：使用 Gemini 2.5 Pro 作为评判模型，按照特定评分细则（如指令遵循、区域定位、源内容保留、视觉质量、时间一致性、IP 身份匹配等）进行打分。

### 4. 资源与算力
论文在附录中给出了详细的训练超参数（如有效全局批次大小为 `128`，混合精度 `bf16`，分布式策略 `ZeRO-2`，两阶段训练的分辨率），但**未明确提及所使用的 GPU 型号、数量及总训练时长**。

### 5. 实验数量与充分性
*   **实验数量**：设计了三组主要实验：1) 在 AgentEdit-Bench 上的主实验对比；2) 在两个已有基准上的性能验证；3) 消融实验（VLM 代理训练阶段、代理跨模型迁移）。
*   **实验充分性**：
    *   多基准、多基线、多对照组（有/无代理）的设计较为全面。
    *   定性结果通过大量图片对比展示，定量结果由强大的 VLM 评判器打分，并提供了详细的评分细则，评估角度多样。
    *   消融实验清晰地展示了 SFT 和 DPO 各自的价值，以及代理的可迁移性。整体实验设计逻辑连贯，评估客观，比较公平。

### 6. 论文的主要结论与发现
*   Aurora 框架通过引入 VLM 代理，显著解决了视频编辑中的欠指定问题。在 AgentEdit-Bench 上，加上代理后，Aurora 的整体得分从 **74.7 提升至 87.9**。
*   即使指令已完备，Aurora 在 EditVerse-Bench 和 OpenVE-Bench 上也表现出强大竞争力，尤其是指令重写功能对所有模型都有普遍提升作用。
*   VLM 代理具有良好的可迁移性，将其输出提供给其他冻结的视频编辑模型（UniVideo, Kiwi-Edit），也能带来稳定的性能增益。

### 7. 优点
*   **创新的问题视角**：将挑战从“模型设计”转移到“条件构建”，抓住了用户交互中的真实痛点。
*   **优雅的解耦设计**：清晰地将代理（负责规划）与视频模型（负责生成）分离，使得两者可以独立进化或替换。
*   **鲁棒的代理训练策略**：结合 SFT 和 DPO，不仅教会代理如何做，还教会其在边界情况下的更优选择，提升了工具使用的准确性。
*   **全面的评估体系**：构建了专门针对欠指定问题的 AgentEdit-Bench，并通过跨模型实验验证了代理模块的通用性，评估方法严谨。

### 8. 不足与局限
*   **训练范式局限**：VLM 代理使用离线 DPO 对齐，而非更优的在线强化学习（如 GRPO），后者通过实际生成视频的反馈来优化代理，可能效果更好，但受限于算力。
*   **模型容量与运动建模**：基于 5B 参数的视频 DiT，在处理大范围、物理上合理的运动插入（如奔跑的动物）时仍有困难，这受限于模型规模和训练数据覆盖度。
*   **潜在风险**：框架中集成的网络图像搜索工具，带来了潜在的版权侵犯、非自愿身份操纵和生成虚假证据的风险。
