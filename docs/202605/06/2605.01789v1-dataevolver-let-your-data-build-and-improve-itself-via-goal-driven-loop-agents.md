## 摘要
可控的视觉数据构建是图像编辑和多模态理解中的主要瓶颈。有效的监督信号很少能通过单次渲染获得；它通常是在迭代生成、检查、校正、过滤和导出过程中逐步形成的。我们提出了 DataEvolver，这是一种闭环视觉数据引擎，将上述过程组织在显式目标、持久化的产物、有界的校正操作以及接受决策的框架下。DataEvolver 支持多种类型的产物，包括 RGB 图像、掩膜、深度图、法线图、网格、姿态、轨迹以及审阅记录。在当前版本中，系统通过两个耦合的循环运行：样本级的生成时自我校正和跨数据集轮次的验证时自我扩展。我们在图像级的物体旋转任务上验证了该框架。使用固定的 Qwen-Edit LoRA 探针，我们的最终模型 Ours+DualGate 在 SpatialEdit 和保留的评估集上均优于未经适配的基础模型和公开的多角度 LoRA。消融实验表明，从场景感知生成到基于反馈的校正，再到双门控验证，该框架展现出一致的性能提升路径。除发布的旋转数据外，我们的主要贡献是提供一个可复用的框架，用于通过显式目标跟踪、审阅、校正和接受循环构建视觉数据集。

## Abstract
Constructing controllable visual data is a major bottleneck for image editing and multimodal understanding. Useful supervision is rarely produced by a single rendering pass; instead it emerges through iterative generation, inspection, correction, filtering, and export. We present DataEvolver, a closed-loop visual data engine that organizes this process around explicit goals, persistent artifacts, bounded corrective actions, and acceptance decisions. DataEvolver supports multiple artifact types, including RGB images, masks, depth maps, normal maps, meshes, poses, trajectories, and review traces. In the current release, the system operates through two coupled loops: generation-time self-correction within each sample and validation-time self-expansion across dataset rounds. We validate the framework on an image-level object-rotation setting. With a fixed Qwen-Edit LoRA probe, our final Ours+DualGate model outperforms both the unadapted base model and a public multi-angle LoRA on SpatialEdit and a held-out evaluation set. Ablations show a consistent improvement path from scene-aware generation to feedback-driven correction and dual-gated validation. Beyond the released rotation data, our main contribution is a reusable framework for building visual datasets through explicit goal tracking, review, correction, and acceptance loops.

  > 数据由目标驱动的循环智能体（goal-driven loop agent）生成、校正、验证并接受。

  - 明确的目标契约（目标变换及保持不变项）；

## 二、提出的方法论

### 1. 核心思想

- **总体框架：DataEvolver 引擎**  
  将数据构建转化为由**目标驱动智能体**执行的闭环过程，包含生成、审查、修正、再生成和接受判定五个阶段。  
  数据的生成和改进由显式目标约束驱动，并在日志、产物和决策层面可追溯。

### 2. 核心结构

- **目标驱动循环体公式：**

  \[
  \text{Skill} = (\text{Stages}, \text{Controller}, \text{Review}, \text{Verdict})
  \]  

  其中：
  - **Stages**：构建阶段及其产物；
  - **Controller**：管理控制逻辑及参数调优；
  - **Review**：VLM/CV 模型审查与诊断；
  - **Verdict**：决策逻辑（接受、再生成、拒绝等）。

- **数据样本的图结构表示：**  

  \[
  G = (S_\text{scene}, A_\text{asset}, P_\text{action}, R_\text{render}, M_\text{geom}, T_\text{temporal}, Q_\text{review}, V_\text{verdict}, E_\text{export})
  \]

  每个样本被组织为带多层产物（图像、几何、轨迹、审查）的**可追溯图谱**。

### 3. 双循环（Dual-Loop）机制
- **内循环（Inner Loop）**：生成时自我校正  
  - 针对单个样本，依据 VLM/CV 信号修正光照、姿态、掩膜等；
  - 调整需要在有界参数范围内进行（报价如重渲染、重定位、材质修正等）。  
- **外循环（Outer Loop）**：验证时自我改进  
  - 依据下游验证结果（任务性能弱点、特定角度表现差等）确定下一轮数据补充；
  - 更新过滤规则或生成策略，实现“数据自增长”。
  
### 4. 实现流程（Data-Build Pipeline）

1. **资产准备（Asset Preparation）**：生成或导入对象素材（SAM3 分割、Hunyuan3D 重构）。
2. **场景搭建（Scene Setup）**：配置光照、相机、场景模板。
3. **变换程序（Action Program）**：定义控制动作（旋转、平移、缩放等）。
4. **多产物渲染（Multi-Artifact Rendering）**：输出 RGB、mask、depth、normal、pose 等。
5. **导出模式（Export Schema）**：依据任务导出图像对、视频序列、轨迹包或诊断日志。

## 三、实验设计

### 1. 任务与数据集设定
- **验证任务**：  
  - *对象旋转任务（Scene-Aware Object Rotation）*。

### 2. 评价基准（Benchmark）
- **外部基准**：SpatialEdit-Bench（488 对外部评估样本）。  
- **内部测试集**：Eval1 Held-out Set（56 对样本）。  
- **探针模型**：Qwen-Image-Edit-2511 LoRA 作为固定编辑模型（仅用于测试数据有效性）。

### 3. 对比方法
- **Base**：原始 Qwen-Image-Edit-2511 模型。  
- **Public LoRA**：fal 提供的多角度 LoRA 公共模型。  
- **Ours 系列**：
  - Ours-Base（基础构建）  
  - Ours+Feedback（外循环反馈）  
  - Ours+InnerGate（加内循环质量门控）  
  - Ours+DualGate（双重门控，全流程闭环）

### 4. 评价指标
- **传统指标**：PSNR、SSIM、LPIPS。  
- **语义/一致性指标**：CLIP-I、DINO、VIE-Score (view + consistency)。
