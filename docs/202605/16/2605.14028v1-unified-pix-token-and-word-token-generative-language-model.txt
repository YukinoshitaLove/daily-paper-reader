## ABSTRACT 

Since the emergence of Vision Transformer (ViT), it has been widely used in generative language model and generative visual model. Especially in the current state-of-art open source multimodal models, ViT obtained by CLIP or SigLIP method serves as the vision encoder backbone to help them acquire visual understanding capabilities. But this method leads to limitations in visual understanding for details, such as difficulty in recognizing small text or numbers in images. To address these issues, we propose a new model to unify pix token and word token into the generative language model. The new model also features with each pix of image having its own token embedding, color folding, global conditional attention approximation and image unsupervised pretraining. We conducted image unsupervised pretraining experiments using our new model to explore its potential. The experimental results show that it has good performance even in small model and with limited training data. We believe our model also conforms to the scaling law, as long as model parameters and training data increased, its performance will continue to improve. 

Keywords Pix Token Embedding · Color Folding · Global Attention Approximation · Image Unsupervised Pretraining 

## 1 Introduction 

Since the emergence of Vision Transformer (ViT), it has been widely used in generative language model and generative visual model. Especially in the current state-of-art open source multimodal models, just like Llava, ViT obtained by CLIP or SigLIP method serves as the vision encoder backbone to help them acquire visual understanding capabilities. But this method leads to some issues. Either there are limitations in visual understanding for details, such as difficulty in recognizing small text or numbers in image. Either there are visual understanding drifts in multi-turns of conversations, the first turn of conversation is correct, but errors are prone to occur later on. Although these limitations can be alleviated by adding more training data related to the issues or optimization CLIP method, there will still be other challenges in the future. We believe that these limitations are likely due to the CLIP method, including the following reasons. 

1) The encoded features cannot represent all the details of the image 

ViT encodes images into a small features that is similar to a text features. But what if this text does not fully describe other small details of the image? Or can the text describe all the details of the image? In this case, the encoded features of the image cannot represent all the details of the image. 

2) The slightest nudge causes the widest chain reaction 

If a small change is made for the image, for example the license plate number of car in the image changes from 1 to 6, the all floating-point numbers in the encoded features will change. The slightest nudge causes the widest chain reaction, which is not good. 

3) ViT use a fake token embedding 

ViT forms a fake token embedding by patching the pixels in the image. But this fake token embedding does not have truly adjustable parameters. So the expressive ability of ViT is questionable. We may need to propose a new architecture to address its shortcomings.

## 2 Related work 

2.1 ViT (Vision Transformer) and application ViT (Vision Transformer) and application [Dosovitskiy et al., 2020] is a computer vision model based on Transformer architecture [Vaswani et al., 2017]. It transforms image processing tasks into sequence processing problems, making breakthrough progress in multiple visual tasks. The core idea of ViT is to view an image as a sequence of image patchs. Specifically, it devides the input image into fixed sized non overlapping small patchs, and then linearly projects each small patch into a token embedding. These token embedding sequences, along with an additional learnable classification token embedding, form the input of the Transformer encoder. 

CLIP (Contrastive Language-Image Pre-training) [Radford et al., 2021] is a multimodal visual language pre training model proposed by OpenAI in 2021. Its core goal is to map images and text to a unified semantic space through contrastive learning, achieving cross modal semantic alignment. 

SigLIP (Sigmoid Loss for Language-Image Pre-training) [Zhai et al., 2023] is a multimodal pre training model proposed by Google DeepMind in 2023. It is an improved version of CLIP, with the core innovation of using Sigmoid loss function instead of traditional Softmax contrastive loss, significantly improving training efficiency and cross modal alignment performance. CLIP and SigLIP are multimodal contrastive learning models based on the Vision Transformer (ViT) architecture, aimed at mapping images and text to the same semantic space for cross modal understanding. In LLM, ViT obtained through CLIP or SigLIP methods is used as the vision encoder backbone to assist large language models in acquiring visual understanding capabilities. For example, LLAVA [Liu et al., 2023]. Currently, excellent open-source multimodal LLMs are further optimized based on the SigLIP method to achieve better image understanding capabilities, such as Kimi-2.5 [Team et al., 2026] and QWen-2.5-VL [Bai et al., 2025].

22.2 Pix Token and unsupervised pretraining 

Research work in this area includes iGPT [Chen et al., 2021], GPT-3 [Brown et al., 2020] and so on. 

iGPT applies Transformers to image pixels after reducing the image resolution and color space. The model is trained as a generative model in an unsupervised manner and can then be fine-tuned or linearly probed on the obtained representations to achieve classification performance, reaching a maximum accuracy of 72% on ImageNet. In iGPT, the method for handling image resolution and color space involves first using the input resolution (IR) approach to reduce the resolution. Then, by applying k-means with k=512 to cluster the (R, G, B) pixel values, the context size is further reduced by a factor of 3, resulting in the model resolution (MR). [Ranzato et al., 2016] applied a similar method to spatial patches. Ultimately, the VQ-VAE [Delaitre et al., 2011] method was employed to downsample the images after MR. The clustering of (R, G, B) values by iGPT can also be interpreted as training a VQ-VAE using an identity encoder and decoder. iGPT ultimately selected a latent vocabulary size of 4096, which is the minimum size without observed reconstruction artifacts. 

GPT-3 builds language understanding ability by learning to predict the next word through unsupervised pretraining on massive Internet texts (about 300B tokens). After unsupervised pretraining, the GPT-3 model can adapt to new tasks with a small number of examples or natural language instructions without fine-tuning through few shot learning and zero shot learning capabilities. The success of GPT-3 has also driven the development of subsequent models such as GPT-3.5 and ChatGPT, becoming an important cornerstone of the generative AI revolution. Unsupervised pretraining and supervised posttraining have become the standard pipelines for training generative models. 

Some differences: iGPT is a generative model that uses unsupervised training and the obtained representation can be fine tuned or linearly probed to obtain classification performance. First Its goal is for image classification, our goal is to unify pix token and word token to generative model and do unsupervised pretraining and supervised posttraining both for pix token and word token. Second, it uses the entire sequence of image pixels for attention operations. Our proposed global attention approximation method avoids using the entire sequence of image pixels for attention operations. The method of processing image resolution and color space in iGPT is equivalent to training an identity encoder and decoder to reconstruct (R, G, B) values. The method of the iGPT is different from the one we proposed called color folding. The color folding method has a simple logic and conforms to the laws of the human eye and nature, without the need for complex and meaningless reconstruction. GPT-3 proves the importance of training with massive text data. CLIP or SigLIP method is only the ViT encoder obtained by training on limited supervised image data. But our model is capable of training on massive unsupervised image data. This laid the foundation for success in visual understanding. 

2.3 Global attention with Transformer 

The Transformer architecture employing full attention mechanisms on full-sized images requires substantial compu-tational resources, which exceed current capabilities and are thus unfeasible. Current research efforts are exploring ways to reduce computational load or avoid direct full-attention mechanisms. Research work in this area includes Swin Transformer [Liu et al., 2021], QWen-2.5-VL and so on. 

Swin Transformer uses shifted window based self-Attention to avoid full attention Swin Transformer, like ViT, divides images into token embeddings based on patches. Then use multi head self attention based on shifted window to these token embeddings. Shifted windows include W-MSA and SW-MSA. W-MSA is a regular window, while SW-MSA is an irregular window. Swin Transformer will also merge token embeddings layer by layer. 

Qwen2.5-VL employs a redesigned Vision Transformer (ViT) architecture as vision encoder to enhance performance and efficiency. Structurally, they incorporate 2D-RoPE and window-based attention mechanisms to support native input resolutions while accelerating the computation of the entire visual encoder. During both training and inference, the height and width of the input images are resized to multiples of 28 before being fed into the ViT. In their architecture, only four layers employ full self-attention, while the remaining layers utilize windowed attention with a maximum window size of 112×112 (corresponding to 8×8 patches). Regions smaller than 112x112 are processed without padding, preserving their original resolution. This design allows the model to operate natively at the input resolution, avoiding unnecessary scaling or distortion. But in the Qwen2.5-VL paper, the architecture figure show that they use M window-based attention blocks in front and the only last block using full attention. 

Some differences: Swin Transformer first uses regular window attention, and then uses inregular window attention. QWen-2.5-VL first used regular window attention, and finally used global attention. ## 3 Method 

The unified pix token and word token model we propose is based on Pix Token Embedding, which we will discuss in section 3.1. In addition, to reduce computational complexity, we propose Color Folding, which will be discussed in section 3.2. We will discuss our unified model in section 3.3. At last we discuss its advantages in section 3.4. 

3.1 Pix Token Embedding 

In the current popular LLMs, if we ignore the fancy tokenizing method of the tokenizer, we can approximately assume that each word token has its own independent token embedding. It is this method that gives each word token sufficient expressive power, and after training with massive text data, its parameters can be adjusted to appropriate positions, thereby improving the ability of LLMs. ViT divides the pixels in the image into patches, and then forms a token embedding by channel for each pixel in each patch. We believe that this approach at least has two drawbacks: First it does not have adjustable parameters, it is a fake token embedding. Second not every pix has its own independent token embedding. Before proposing our method, we make the following assumptions: The shape of image with 3 color channels (RGB channels) is (3, H, W), where 3 is the number of channels, H is the image height, and W is the image width. And the length of the color channel is L which defaults to 256. We believe that every pix should also have its own true token embedding. To achieve this goal, our design is as follows: Each pix of image corresponds to a unique one token which has one’s own token embedding. As shown in Figure 1. The value of pix token is related to the color value of the pix. The formula for calculating the value of token is: 4T oken value = r + g ∗ L + b ∗ L ∗ L

L is the length of the color channel. r is R channel value and g is the G channel value and b is the B channel value. If L is 256 , the token value range is [0 , 256 ∗ 256 ∗ 256] . So the total number of tokens dictionary is 256 ∗ 256 ∗ 256 .Because every token have it’s own token embedding, so the total number of token embeddings is 256 ∗ 256 ∗ 256 . The dimension of token embedding is set according to the actual situation, and we define it as d. In space, these token embeddings are continuous and the tensor shape is ( 256 ∗ 256 ∗ 256 , d). Attention: Actually also there is an additional pad token for pix due to differences in height and width of image. 

3.1.1 Pix Token Embeddings Sequence 

Next, we will investigate how to obtain a token embeddings sequence from all the pixels in the image. Our combination method is sequential: In order from left to right, and then from top to bottom, extract a pix token from the image and find its corresponding pix token embedding. Then combine these pix token embeddings in chronological order to form a token embeddings sequence. Taking an image with L= 256 and height H and width W both being 3 as an example, as shown in Figure 1: The first pix p1 is (20 , 8, 2) , its token value is equal to 2 ∗ 256 ∗ 256 + 8 ∗ 256 + 20 = 133140 .The 7th pix p7 is (3 , 9, 128) , its token value is equal to 128 ∗ 256 ∗ 256 + 9 ∗ 256 + 3 = 8390915 .Similarly, we obtain a pix token sequence [p1, p 2, p 3, p 4, p 5, p 6, p 7, p 8, p 9].Then based on this pix token sequence, we get a pix token embedding sequence from the token embedding dictionary. 

3.1.2 Lossless of Data Preprocessing 

As is well known, image data needs to be preprocessed before model training. The traditional method is to map RGB color values from [0 − 255] to [0 − 1.0] , for example 1/255 = 0 .00392156862745098 . Due to the limited number of floating-point digits in tensors, it is necessary to perform bit truncation on color values, which can result in losses. Although we think the loss value is relatively small, considering the lower learning rate and different optimizer methods and considering training with 16 bits or lower, there are imperfections. The Pix Token Embedding method we propose will not have this problem, it is lossless data preprocessing.

3.3.1 Global to Local Window 

Assuming that the shape of pix token embedding sequence obtained from image (3, H, W) according to the method in section 3.1.1 is (N,d), N=H*W is the sum of all pixels in the image. We divide the image into (N/K)**2 windows in order from left to right and top to bottom and each window the height and width are both K. The pix token embedding of pixels in each window forms a pix token embeddings sequence. The pix token embedding sequence of all window forms a batch. So the pix token embedding sequence dimension changes from (N, d) to (N/K**2, K**2, d). In this way, The global pix token embedding sequence becoming a batch local window pix token embedding sequence. 8Taking Figure 5 as an example to illustrate: The global pix token embedding sequence changes from (16, d) to (4, 4, d). Both H and W are 4, N is 16 and K is 2. Note: W must can be divided by the K. If the H cannot be divided by the K, it needs to be filled with pix pad token to the same long as W. 

3.3.2 Local Window Conditional Attention 

We received a batch of local window pix token embedding sequence on section 3.3.1, dimension is (N/K ∗∗ 2, K ∗∗ 2, d ).Then perform mask multi-head self attention operation on this batch sequence. Position Embedding: We change the local window pix token embedding sequence to global sequence and add position embedding with a rotary embedding way. After that we change to local window pix token embedding sequence again. Local Window Conditional Attention: Perform mask multi-head self attention operation on K**2 pix token embeddings according to N/K**2 batches. To illustrate with Figure 5: The first window, perform mask multi-head self attention on the sequence (P1, P 2, P 5, P 6).The secnd window, perform mask multi-head self attention on the sequence (P3, P 4, P 7, P 8).The third window, perform mask multi-head self attention on the sequence (P9, P 10 , P 13 , P 14 ).The forth window, perform mask multi-head self attention on the sequence (P11 , P 12 , P 15 , P 16 ).

3.3.3 Get Last Token Embedding 

We get the last pix token embedding from every local window of the final output after local window attention. The output dimension is (N/K**2, d). To illustrate with Figure 5: We get a token embedding sequence (E6, E 8, E 14 , E 16 ).The probabilitie of E6 is P (E6|P1, P 2, P 5, P 6).The probabilitie of E8 is P (E8|P3, P 4, P 7, P 8).The probabilitie of E14 is P (E14 |P9, P 10 , P 13 , P 14 ).The probabilitie of E16 is P (E16 |P11 , P 12 , P 15 , P 16 ).

Global Conditional Attention Approximation: 

Observing the entire process, we use a clever method to approximate global conditional attention. The reasons are as follows, refer to Figure 4: Assuming the input text word token is X, the input image pix tokens is (p1, p 2, ..., pN ), and the output re-sponse of model is Y . The probability of Y is P (Y |X, p 1, p 2, ..., p N ). But due to the length of pix tokens be-ing too long, for example the length of pix tokens of image with a height and width of 256 is 65556, It’s not feasible to do self attention operation. Our method can avoid this problem, and approximate the goal. We know (Ek, E 2k, ..., E N ) are the token embedding after local window conditional attention. The probabil-ity of Ek is P (Ek|p1, p 2, ..., p K ). The probability of E2k is P (E2k|pK+1 , p K+2 , ..., p 2K ). The probability of 

EN is P (EN |pN −K+1 , p N −K+2 , ..., p N ). The probability of Y is P (Y |X, E k, E 2k, ..., E N ). we obtain the equa-tion: P (Y |X, E k, E 2k, ..., E N ) ≈ P (Y |X, p 1, p 2, ..., p K , p K+1 , p K+2 , ..., p 2K , ..., p N −K+1 , p N −K+2 , ..., p N ) =

In the unified token space, we can do self attention operation and token predict for both pix token and word token. But in Llava, we cannot do pix token predict. This is not perfect. 

3.4.3 Say bye bye to "the slightest nudge causes the widest chain reaction" 

We know that image is encoded as features by ViT in CLIP method. But if a small change is made for the image, for example the license plate number of car in the image changes from 1 to 6, the all floating-point numbers in the encoded features will change. The slightest nudge causes the widest chain reaction, which is not good. Our model can avoid such problems. Only one or a few token embedding have undergone changes, but most of the others will not change. This makes it easier for model to capture small changes. 

3.4.4 Image unsupervised pretraining 

GPT-3 tells us that extensive word token unsupervised pretraining is the cornerstone for LLMs to achieve success. But CLIP method is using image supervised training. This is clearly the reason for the missing distribution. Our model finally makes image unsupervised pretraining possible. Our model can undergo the following pretraining: 

1) Only word token pretraining 

This is the same as the currently popular method. We won’t go into detail anymore. 

2) Only pix token pretraining 

We make the first pix token predict of next local window for the input image. Assuming the token starting symbol and ending symbol is <s> and </s>, the image starting symbol and ending symbol is <img> and </img>. Pix token predict sequence is (p1, p k+1 , p 2k+1 , ..., p ik +1 , </img> )

The p1 token probability is: P (p1|<s> , <img> ).The pk+1 token probability is: P (pk+1 |<s> , <img> , E k).The p2k+1 token probability is: P (p2k+1 |<s> , <img> , E k, E 2k)

... The pik +1 token probability is: P (pik +1 |<s> , <img> , E k, E 2k, E ik )

The </img> token probability is: P (</img> |<s> , <img> , E k, E 2k, ..., E N/K ∗∗ 2)

pik +1 is the first pix token of the i + 1 local window, i = [0 , N/K ∗ ∗ 2] .

3) Mixed word token and pix token pretraining 

There are massive mixed samples of text and image in reality world, for example, image news on the Internet. We can now use these mixed sample for pretraining. This is something that previous methods like Llava were unable to achieve! But how to deal with batch datas in pretraining? Our model encapsulate the operations (steps 2, 3, 4, and 5 in section 3.3) on image into a module called ImageBlock. It accepts input from batch images and its output is batch pix token embedding sequence. Taking 3 samples as an example: sample S1 is only text: (T). sample S2 is text and image mixture: (I, T). sample S3 is text and image mixture: (T1, I, T2). T is a text sequence, I is a shape of (3,H,W) image. Brief pretraining process: 1) First obtain T of all samples and get word token embedding sequence and label tokens. 2) Second obtain I of all samples and its label tokens, then send it to the ImageBlock to obtain pix token embedding sequence. 10 3) Connect word token embedding sequence and pix token embedding sequence into a sequence. Connect label tokens of T and I into a sequence. 4) Perform unified conditional attention operation on the connected token embedding sequence and final output predict tokens. 5) Finally, the obtained predict tokens and label tokens will undergo Cross-Entropy Loss operation. 

## 4 Experiments 

We conducted a only image unsupervised pretraining experiments using our unified pix token and word token model. The purpose of the experiment is to demonstrate the feasibility of our model, and we expect its results to be similar to those of only text unsupervised pretraining like GPT-3. Due to our lack of computing resources, we are unable to conduct other experiments, such as mixed text and image unsupervised pretraining. We hope everyone can understand. So we conducted the experiment that best demonstrated the potential of our model, that is only image unsupervised pretraining. 

4.1 Settings 1) Dataset 

Training dataset: Due to our lack of computing resources, we used a dataset with a small quantity and low resolution. We used one hundred thousand samples from the llava-cc3m-pretrain-595k dataset, about 5 billion pix tokens. Validate dataset: We used one thousand samples from the llava-cc3m-pretrain-595k dataset. 

2) Model 

Due to our lack of computing resources, we used a model with 120 millions parameters. The configuration information of the model is as Table 1. 

3) Training 

We trained our model, using learning rate 0.0006, cosine learning rate scheduler, AdamW with β1 = 0.9, β2 = 0.999, dropout of 0.1, weight decay of 0.01, gradient clipping at global norm 1, batch size 2. Image related parameters: image size is 224, color fold factor is 16, and local window size is 16. 

4.2 Experimental Results 

The training loss curve and validation loss curve are shown in the Figure 6. The training loss curve converges with the number of elapsed tokens, which is similar to the only word token pretraining of GPT-3. But compared to pretraining of GPT-3, it converges faster in a relatively small number of pix tokens. The validation loss curve also converges like the training loss curve. This proves the feasibility of our unified token model. 

Experimental conclusion: 

Our model with only 120 millions parameters can achieve good results with limited training data. The effect should be better if we use a tens of billions parameters model. If more massive training data is used, model will inevitably grasp the patterns of the pix token sequence. In short, we believe that when we increase the number of parameters and datas, our model also conforms to the scaling law of generative language model.

We first analyzed the limitations of current state-of-art open source generative language models in visual understanding. Our model have many advantages, including the ability to undergo image unsupervised pretraining. We conducted only image unsupervised pretraining experiments using our model. The experimental results have proven that after a certain amount of data pretraining, our model can learn the patterns of the pix token sequence. We believe that when we increase the number of parameters and datas, our model also conforms to the scaling law of generative language model. The new model we proposed is a perfect way to unify pix token and word token together into generative language model.
