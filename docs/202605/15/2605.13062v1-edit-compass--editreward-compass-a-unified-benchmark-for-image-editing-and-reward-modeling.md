## 摘要
近年来，图像编辑模型在指令遵循、多模态理解以及复杂视觉编辑方面取得了显著进展。然而，现有基准由于任务难度有限和评估协议过于粗粒度，往往无法准确反映人类判断，尤其对于前沿强模型更为明显。同时，基于强化学习（RL）的图像编辑优化中，奖励模型的重要性日益凸显，但现有奖励模型基准仍依赖于脱离实际RL场景的不切实际评估设置。这些局限性阻碍了对图像编辑模型及奖励模型的可靠评估。为了解决这些问题，我们提出了 Edit-Compass 和 EditReward-Compass，一个用于图像编辑与奖励建模的统一评估套件。Edit-Compass 包含 2,388 个精心标注的实例，涵盖六个逐步增加难度的任务类别，覆盖诸如世界知识推理、视觉推理和多图像编辑等能力。除了广泛的任务覆盖之外，Edit-Compass 还采用基于结构化推理和精心设计评分标准的细粒度多维评估框架。同时，EditReward-Compass 包含 2,251 对偏好样本，模拟强化学习优化过程中真实的奖励建模场景。

## Abstract
Recent image editing models have achieved remarkable progress in instruction following, multimodal understanding, and complex visual editing. However, existing benchmarks often fail to faithfully reflect human judgment, especially for strong frontier models, due to limited task difficulty and coarse-grained evaluation protocols. In parallel, reward models have become increasingly important for RL-based image editing optimization, yet existing reward model benchmarks still rely on unrealistic evaluation settings that deviate from practical RL scenarios. These limitations hinder reliable assessment of both image editing models and reward models. To address these challenges, we introduce Edit-Compass and EditReward-Compass, a unified evaluation suite for image editing and reward modeling. Edit-Compass contains 2,388 carefully annotated instances spanning six progressively challenging task categories, covering capabilities such as world knowledge reasoning, visual reasoning, and multi-image editing. Beyond broad task coverage, Edit-Compass adopts a fine-grained multidimensional evaluation framework based on structured reasoning and carefully designed scoring rubrics. In parallel, EditReward-Compass contains 2,251 preference pairs that simulate realistic reward modeling scenarios during RL optimization.

- **核心问题**：如何构建一个兼具**高任务复杂性、细粒度评估维度**、并与**人类判断一致**的统一评测体系，用以同时衡量**图像编辑模型性能**和**奖励模型效能**。
- **研究目标**：提出一个覆盖多任务类别、支持RL奖励评估、并具备人类一致性标准的**统一基准套件**。

## 二、方法论与核心设计

### 1. 统一方案概览
论文提出两个互补性部分：
- **Edit-Compass**：用于图像编辑模型的多维任务评测；
- **EditReward-Compass**：用于奖励模型的偏好判断与强化学习一致性测试。

两者共同构成一个从编辑能力到反馈评估的闭环体系。

### 2. Edit-Compass 的设计思路
- **数据规模与任务结构**：  
  包含 **2,388** 个高质量样本，分为 **6 大类、36 个子任务**，涵盖从基础修改到复杂推理：
  1. General Editing（通用编辑）  
  2. Dynamic Manipulation（动态操作）  
  3. World Knowledge Reasoning（世界知识推理）  
  4. Algorithmic Visual Reasoning（算法视觉推理）  
  5. Multi-Image Editing（多图像理解与编辑）  
  6. Complex Tasks（复合指令与视觉标注任务）
- **数据来源**：  
  由专家与 Gemini 3 Pro、GPT-5.1 生成并经人工审核；某些推理类任务由程序自动生成带有算法真值的图像。
- **评估维度与指标体系**：  
  三个核心维度：
  - *Instruction Awareness (IA)*：指令理解与执行正确性；
  - *Visual Consistency (VC)*：非编辑区域保持一致性与身份连续性；
  - *Visual Quality (VQ)*：生成图像的自然性、结构连贯与无伪影。
- **评分框架**：  
  利用多模态大模型（Gemini 3.1 Pro）作为评审代理，基于多级推理与 JSON 结构化输出进行自动评分，刻度为 **1–5分**，并采用维度加权的几何平均（根据任务类型调整权重）。

### 3. EditReward-Compass 的设计思路
- **数据组成**：  
  包含 **2,251 对偏好样本对 (preference pairs)**，每对含：
  - 编辑指令；
  - 两个候选编辑结果；
  - 人类专家一致标注的偏好结果。
- **构造方式**：
  - 基于 Edit-Compass 的任务指令；
  - 通过多模型采样获得候选输出，模拟 RL 训练中的候选集（包含 FlowGRPO 风格随机噪声采样策略）；
  - 八位专家两阶段投票筛选，确保一致性（5/5 一致才保留）。
- **评估维度**：与 Edit-Compass 一致（IA、VC、VQ），实现编辑模型与奖励模型之间的统一比较。

### 4. 技术设计与算法流程（文字化）
1. **数据采样与生成** → 三种模式（人工选取、AI增强生成、程序合成）；
2. **任务模板构建** → 每类任务对应固定指令模式；
3. **模型推理评审** → 多模态评审模型执行结构化推理；
4. **多维评分汇总** → 通过加权几何平均生成总体得分；
5. **偏好对评估** → 奖励模型根据相同维度输出一致性判断。

### 2. 对比对象

  - *奖励模型*：Qwen3-VL、Gemma3/4、GPT-4.1、EditScore、EditReward 各版本等。

### 3. 指标体系与实验内容
- 对每个模型计算 IA、VC、VQ 三项指标；
- 分语言（英文指令 / 中文指令）两套完整评测；
- 额外实验包括：
  - 不同任务子类的性能分布；

## 六、主要结论与发现

- **性能差距显著**：
  - 最强闭源模型（Nano Banana Pro）总体得分 ≈ **3.99**；
  - 最强开源模型（Qwen-Image-Edit）约 **2.69**；
  - 最大差距体现在“世界知识推理”和“多图像编辑”任务。

- **人类一致性提升**：
  - Edit-Compass 在人工偏好一致性指标上明显超过旧基准。

## 七、研究亮点与创新

- **统一跨模态评测框架**：首次将图像编辑任务与奖励建模任务纳入同一评测体系；

- **逼近真实RL训练分布的采样策略**：EditReward-Compass 模拟 FlowGRPO 式策略，有助于评估奖励模型在RL场景下的真实性能；
- **双语设计**：同时提供中英任务，考察模型跨语言编辑一致性。

## 八、不足与局限

  视频或三维编辑场景未完全纳入；
- **长期适配问题**：  
  随着LLM迭代，基准必须持续更新以保持可比性；
- **奖励建模指标仍基于偏好对**：  
  尚未验证自动化reward尺度在连续RL过程中的稳定性。
