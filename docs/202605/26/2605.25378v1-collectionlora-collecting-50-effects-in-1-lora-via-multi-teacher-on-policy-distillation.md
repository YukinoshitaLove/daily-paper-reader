## 摘要
定制化图像编辑旨在利用少量配对数据为预训练扩散模型赋予特定视觉效果，通常通过低秩适应（LoRA）实现。随着所需效果数量的增加，存储和动态加载众多效果LoRA显著增加了部署开销。此外，当前流程通常将这些效果LoRA与加速模块级联以实现快速生成，这会引发严重的参数干扰，导致概念混合和风格退化。我们提出CollectionLoRA，一个多教师在线策略蒸馏框架，能够将多达50种不同效果LoRA的概念及少步生成能力蒸馏到单个LoRA中，从根本上解决了特征干扰问题，并大幅降低部署成本。具体方法引入了：(i) 概率双流路由机制，使模型在训练过程中能够随机切换数据源，有效增强其对未见场景的泛化能力；(ii) 非对称正交提示策略，在提示空间内实现概念隔离；(iii) 由粗到精的蒸馏目标，以缓解教师模型与学生模型之间的分布差距。大量评估表明，CollectionLoRA将所有定制效果和少步生成蒸馏到单一LoRA中，减少了部署开销，同时实现了与独立训练的教师模型相当甚至更好的概念保真度。

## Abstract
Customized image editing aims to equip pre-trained diffusion models with specific visual effects using limited paired data, typically via Low-Rank Adaptation (LoRA). As the number of desired effects grows, storing and dynamically loading numerous these effect LoRAs significantly increases deployment overhead. Furthermore, current pipelines typically cascade these effect LoRAs with acceleration modules for fast generation, which triggers severe parameter interference and results in concept bleeding and style degradation. We propose CollectionLoRA, a multi-teacher on-policy distillation framework capable of distilling the concepts of up to 50 different effect LoRAs along with few-step generation capabilities into a single LoRA. This fundamentally resolves the feature interference issue and significantly reduces deployment costs. Specifically, the method introduces (i) a Probabilistic Dual-Stream Routing mechanism that enables the model to randomly switch between data sources during training, effectively enhancing its generalization in unseen scenarios; (ii) an Asymmetric Orthogonal Prompting strategy to achieve concept isolation within the prompt space; (iii) a Coarse-to-Fine Distillation Objective to mitigate the distribution gap between the teacher and student models. Extensive evaluations show that CollectionLoRA distills all customized effects and few-step generation into a single LoRA, reducing deployment overhead while achieving concept fidelity comparable to or better than independently trained teacher models.

## 1. 论文的核心问题与整体含义（研究动机和背景）

在定制化图像编辑领域，通常需要为每一种特定视觉效果训练一个独立的低秩适应（LoRA）模块，并与基础模型及加速 LoRA 级联以实现快速生成。然而，随着效果的增多，这一范式暴露出三个关键瓶颈：

- **存储开销**：每一个效果 LoRA 都需要独立存储，在终端设备上部署时成本剧增。
- **路由延迟与错误**：推理时需要根据指令从 LoRA 库中检索并动态加载对应的模块，带来延迟和匹配失误的风险。
- **参数冲突**：将效果 LoRA 与加速 LoRA 线性叠加会扰乱原始特征流形，导致概念混合（概念出血）和风格退化。

因此，论文的核心目标是**将大量异构视觉效果与少步生成能力整合至一个统一的 LoRA 中**，从根本上消除上述部署与干扰问题。这一任务本质上是一个**多教师向单学生的蒸馏问题**，但在扩散模型、少样本、多概念条件下的蒸馏存在训练不稳定、概念冲突和分布坍塌等严重挑战。

## 2. 论文提出的方法论

CollectionLoRA 是一个**多教师在线策略蒸馏框架**，以 Distribution Matching Distillation (DMD) 为基础，通过三个关键组件实现稳定、高效的多概念融合与少步生成。

### 2.1 概率双流路由（Probabilistic Dual-Stream Routing, PDSR）

- 训练时以概率 \( p_{\text{switch}} \) 随机切换至两条数据流之一：
  - **通用流（General Stream）**：使用未标注的通用图像，以冻结的基础模型作为教师，通过反向模拟（Backward Simulation）计算标准的 DMD 损失，旨在保持基础泛化能力。
  - **效果流（Effect Stream）**：动态加载某一效果 LoRA 实例化为教师，采用专有的 **由粗到精蒸馏目标（C2F-DO）** 注入该效果的专属概念。
- PDSR 相当于一种**结构正则化**，防止模型在少量效果数据上过拟合而丧失对未见场景的泛化能力。

### 2.2 非对称正交提示（Asymmetric Orthogonal Prompting, AOP）

- **教师端**继续使用原始训练提示 \(c^i_{\text{teacher}}\) 生成高质量的参考图像 \(y\)。
- **学生端**则使用由 VLM 自动重写的描述性提示 \(c^i_{\text{vlm}}\)，并附加一个**正交触发词** \(v_i\)，使每个效果在提示空间中占据独立且正交的子空间，从而实现概念隔离，无需人工设计提示。
- 最终学生条件为：\(c^i_{\text{student}} = [v_i, c^i_{\text{vlm}}]\)。

### 2.3 由粗到精蒸馏目标（Coarse-to-Fine Distillation Objective, C2F-DO）

针对初期学生与教师分布差距大、标准 DMD 容易导致分布坍塌的问题，C2F-DO 结合了两种监督信号：

- **轨迹锚定（TA‑FM）**：采用流匹配损失 \(\mathcal{L}_{\text{TA-FM}} = \| G_\theta(y_t, t, c_\text{student}) - (y - \epsilon) \|_2^2\)，为学生提供指向目标图像的稳定优化方向，在早期训练中构建结构基础。
- **目标仿真分布匹配（TS‑DMD）**：在目标图像 \(y\) 上进行仿真，先将其扩散到较浅的时间步 \(t_{\text{gen}} < \tau_{\max}\)，再由生成器去噪得到模拟输出 \(\hat{y}\)，然后重新加噪至较深的时间步 \(t_{\text{critic}} > \tau_{\min}\) 进行评分函数对齐。这一分支强制学生捕获统计方差，恢复高频细节。
  - 严格的**时间步约束**放大了真实评分与虚假评分的差异，确保梯度信号可靠。
- 效果流中还包含 **反向仿真分布匹配（BS‑DMD）** 作为持续性正则化，保持全局风格分布。
- 总优化目标：
  \[
  \mathcal{L}_{\text{C2F-DO}} = \mathcal{L}_{\text{TA-FM}} + \mathcal{L}_{\text{DMD\_TS}} + \mathcal{L}_{\text{DMD\_BS}}
  \]
  最终由 PDSR 动态决定每一步计算 \(\mathcal{L}_{\text{DMD\_BS}}\)（通用流）还是 \(\mathcal{L}_{\text{C2F-DO}}\)（效果流）。

### 2.4 训练与推理流程

- 学生 LoRA \(\Delta\theta_{\text{student}}\) 与虚假评分模型均仅通过 LoRA 微调，学习率 1e-4，生成器训练 5000 步。
- 推理时，仅需加载基础模型与单一 CollectionLoRA，无需任何附加路由或模块组合：
  \[
  x_g = G_\theta(\epsilon, c_{\text{student}}), \quad \theta = \theta_{\text{base}} + \Delta\theta_{\text{student}}
  \]

## 3. 实验设计

- **数据集与基准**：
  - 训练集：内部构建的特效数据集，包含 **50 种效果**（每种约 20 对动物/肖像图像），另有一个 20K 未标注通用图像数据集（配合 MLLM 生成的指令）。
  - 评估集：**EffectBench**，基于 Gemini-2.5 Pro 和 Qwen-Image 生成的两类（动物与肖像）各 100 张高多样性测试图像，共计 **5000 条编辑指令**。
- **对比方法**：
  - `Base`：基础模型（Qwen-Image-Edit-2509）单效果推理（NFE=40×2）。
  - `Base + Lightning`：基础模型 + 加速 LoRA（lightx2v，NFE=8）。
  - `50 in 1 (FM) + Lightning`：聚合全部数据用标准流匹配联合训练的单一 LoRA + 加速模块。
  - `Ours`：CollectionLoRA 蒸馏的单一 LoRA（NFE=8）。
- **评价指标**：

## 6. 论文的主要结论与发现

- CollectionLoRA 成功将 **50 种定制视觉效果及少步生成能力融入单个 LoRA**，推理步数从 80 降至 8 步，大幅降低部署开销。

- PDSR 引入通用数据作为正则化，使模型在极端概念压缩下仍能保持对**未见主体和姿态的泛化能力**。
- 无需额外训练即展现出**零样本效果组合能力**，可通过链式描述同时调用两种效果。
- 框架可扩展至 180 个效果，存储开销仅为传统范式的 2%，且无灾难性质量下降。

## 8. 不足与局限

- **训练计算需求**：尽管未详细给出训练时长，但 8 卡 H800 和 5000 步训练仍有一定资源要求；且需要预先训练好所有单效果教师 LoRA。
- **效果规模上限**：实验表明将效果数量从 50 扩展到 180 时，CLIP 分数从 0.727 小幅下降至 0.709，且在高数量下仍需回退到 VLM 路由（虽然开销仍远低于传统方式），说明压缩极限下性能仍有衰减。
- **模型架构依赖**：实验基于特定的基础模型 Qwen-Image-Edit-2509，未探讨对其他扩散模型（如 Stable Diffusion 3、FLUX）的泛化性。
- **数据集内部使用**：效果数据为自建内部数据集，可能影响结果的可复现性；EffectBench 也是基于同一数据分布构建，存在一定内生性偏差。
- **零样本组合的深度未充分挖掘**：论文展示了组合两个效果的案例，但对更多效果（≥3）的组合、某些效果的互斥性等未做深入分析。
- **人类主观评估规模有限**：用户研究样本量相对较小（50组），未来可扩大规模并引入更多样化的评估者。
