## 摘要
粗掩膜局部图像编辑要求模型在修改用户指定区域的同时保持周围场景不变。然而在实践中，粗略的掩膜往往会成为意外的形状先验：掩膜不仅没有作为灵活的编辑支持，反而将生成的对象拉向其偶然的边界。我们将这种失败现象定义为掩膜形状偏差（mask-shape bias），并通过“两区约束”（Two-Zone Constraint）重新框定该任务——背景应保持稳定，而可编辑区域应遵循指令而不被迫继承掩膜轮廓。BRIDGE 通过将掩膜保持在 DiT 主干之外以进行支持构建和融合，避免了 DiT 内部的掩膜注入和复制控制分支，从而应对这一问题。它采用 BridgePath 生成策略：主路径（Main Path）保留背景上下文，主体路径（Subject Path）从独立噪声中生成可编辑内容。受到 Qwen-Image 诊断实验的启发（该实验表明位置嵌入和注意力连接调节了视觉 token 对图像上下文的重用方式），BRIDGE 引入了可学习的离散几何门（Discrete Geometric Gate），用于 token 级别的位置嵌入路由。该门允许主体 token 在融合区域附近借用以背景为锚的坐标，或保持以主体为中心的坐标以获得几何自由度。我们在 BRIDGE-Bench、MagicBrush 和 ICE-Bench 上评估了 BRIDGE。在 BRIDGE-Bench 上，BRIDGE 将 Local SigLIP2-T 从使用 FLUX.1-Fill 时的 0.262 和使用 ACE++ 时的 0.390 提升至 0.503，同时在局部 DINO 和 DreamSim 指标上也有相应提升。在 MagicBrush 和 ICE-Bench 上的零样本结果进一步表明，BRIDGE 在超出精心策划基准的情境下仍具有较强的指令对齐与源保持能力，而新增的路由模块在参数量上仅为 13.31M，比基于 ControlNet 的复制分支更加紧凑。

## Abstract
Coarse-mask local image editing asks a model to modify a user-indicated region while preserving the surrounding scene. In practice, however, rough masks often become unintended shape priors: instead of serving as flexible edit support, the mask can pull the generated object toward its accidental boundary. We study this failure as mask-shape bias and frame the task through a Two-Zone Constraint, where the background should remain stable while the editable region should follow the instruction without being forced to inherit the mask contour. BRIDGE addresses this setting by keeping masks outside the DiT backbone for support construction and blending, avoiding DiT-internal mask injection and copied control branches. It uses BridgePath generation, where a Main Path preserves background context and a Subject Path generates editable content from independent noise. Motivated by a diagnostic Qwen-Image experiment showing that positional embeddings and attention connectivity regulate which image context visual tokens reuse, BRIDGE introduces a learnable Discrete Geometric Gate for token-level positional-embedding routing. This gate lets subject tokens borrow background-anchored coordinates near fusion regions or keep subject-centric coordinates for geometric freedom. We evaluate BRIDGE on BRIDGE-Bench, MagicBrush, and ICE-Bench. On BRIDGE-Bench, BRIDGE improves Local SigLIP2-T from 0.262 with FLUX.1-Fill and 0.390 with ACE++ to 0.503, with parallel gains in local DINO and DreamSim. Zero-shot results on MagicBrush and ICE-Bench further indicate competitive alignment and source preservation beyond the curated benchmark, while the added routing module remains compact at 13.31M parameters compared with ControlNet-style copied branches.

## 一、研究背景与核心问题

- **研究动机**：在真实交互式图像编辑中，用户通常提供的是粗糙掩膜（coarse mask）——例如简单涂抹或矩形框，而非精确分割边界。  
- **主要问题**：现有基于扩散模型（Diffusion Models）的局部编辑方法在接收掩膜时往往将其视为几何约束，导致生成物体受掩膜形状影响，出现所谓的 **mask-shape bias（掩膜形状偏差）**。
- **核心挑战**：如何既保持背景一致性（不破坏场景布局），又允许生成的新主体在形状上摆脱掩膜约束，实现**局部自由的内容编辑**。
- **研究目标**：提出一种能够解耦“编辑区域定位”与“几何内容生成”的方法，在粗掩膜条件下完成语义一致、几何自然、背景无痕的编辑。

## 二、方法论与技术细节

### 1. 整体框架

- **方法名称**：BRIDGE（Background Routing and Isolated Discrete Gating）  
- **核心思想**：将“掩膜的定位作用”与“生成几何”的内部表征解耦，通过位置嵌入路由控制，使局部编辑时主体（Subject）与背景（Background）协调融合但又相互独立。

### 2. BridgePath 双路径生成架构

- **Main Path（主路径）**：接收原始图像的潜空间表示，用以保持场景结构和背景纹理；
- **Subject Path（主体路径）**：在掩膜对应的轴对齐区域（bounding box）上以**独立高斯噪声**初始化，生成可编辑内容；
- 两路径在每层 DiT（Diffusion Transformer）中共同前向传播，并通过**位置嵌入与注意力路由**进行信息交互。

### 3. Discrete Geometric Gate（离散几何门）

- **作用**：决定每个主体 token 是否使用背景锚定坐标（用于融合）或主体自中心坐标（用于几何自由）；
- **形式化**：  
  - 对于每个 DiT 层 l、主体 token i，预测概率  
    \( p_{li} = \sigma(h_\phi^l(H^l_i)) \)，其中 \( H^l \) 为主从拼接特征；
  - 以 0.5 阈值二值化 \( G_{li} = round(p_{li}) \)；
  - 有效嵌入为  
    \( PE_{eff,i}^l = G_{li} PE_{base,i} + (1 - G_{li}) PE_{swap,i} \)；
  - 使用 Straight-Through Estimator 实现可导二值传导。
- **参数规模**：GateBlocks 共 13.31M 个参数，远小于 ControlNet 复制分支（约 1.13B）。

### 4. 训练目标与流程

- **优化目标**：Flow Matching 形式的速度场预测损失  
  \[
  L_{fm} = \mathbb{E}\|v_\theta(Z_t, T, t, z_{edit}) - (Z_1 - Z_0)\|^2
  \]
  其中 \( Z_t = tZ_1 + (1 - t)Z_0 \)。
- **训练策略**：  
  - 对 Qwen-Image [29] 预训练 DiT 主干施加 rank-512 LoRA；
  - GateBlocks 从零初始化并联合端到端训练；
  - 随机 10% 概率丢弃文本用于 Classifier-Free Guidance。
- **掩膜扰动与数据增强**：对掩膜进行形态学扰动，以打破轮廓依赖，使模型学习将掩膜仅作为定位提示。

## 八、不足与局限

- **计算成本仍高**：Subject Path 增加约 30–40% 延迟和显存；
- **不完全理论化**：属于经验方法，未提供严格数学证明；
- **掩膜边界控制精度有限**：对于不规则用户掩膜仍需外部 blending 调节；
- **数据来源偏重合成场景**：真实交互掩膜的鲁棒性需进一步验证；
- **未来方向**：
  - 动态 token 裁剪与“Packed Latents”以降低开销；
  - 更高质量的真实用户编辑数据集；
  - 自适应边界融合策略。
